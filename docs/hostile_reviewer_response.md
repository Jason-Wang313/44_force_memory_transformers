# Hostile Reviewer Response

The strongest v2 hostile review was correct: the learned forget gate is not the key novelty when a clean reset rule solves the toy. The v3 paper answers by reframing the claim around force-memory lifecycle management and by including hard reset, hysteresis reset, noisy-boundary reset, learned reset classifier, and oracle lifecycle baselines.

The second hostile review is that overall sequence accuracy hides the mechanism. The full-scale suite reports switch accuracy, stale-memory error, missed resets, false resets, boundary F1, retention calibration, sequence success, and force overshoot.

The final claim is therefore diagnostic: contact force memory should be evaluated as managed state whose reset and retention errors are visible.
